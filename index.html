Engineering

Tech stack & system design

A breakdown of what powers CivicAI under the surface — built for clarity, not just a demo.

Frontend
  • Semantic HTML5 + CSS3 (custom properties, no framework lock-in)
  • Responsive grid/flex layout — mobile, tablet, desktop breakpoints
  • Vanilla JS — DOM event delegation, no heavy dependencies
  • Glassmorphism + dark-mode design system (Sora / Inter / JetBrains Mono)
AI / Agent Layer
  • IBM watsonx Orchestrate — agent hosting & conversation runtime
  • Natural language intent classification (issue type, severity)
  • Orchestration ID + Agent ID bound via secure config object
  • Deployed on IBM Cloud (us-south region)
Data & Sync
  • postMessage-based event bridge between agent iframe and UI
  • Client-side priority scoring (CRITICAL / HIGH / MEDIUM / LOW)
  • Live activity feed — DOM diffing, capped buffer (last 8 events)
  • Department routing map (Roads, Water, Electricity, Sanitation)
Accessibility
  • Multi-language ready (Hindi · English · Urdu · Regional)
  • prefers-reduced-motion support for animations
  • Focus-visible states for keyboard navigation
  • High-contrast dark theme tuned for low-literacy readability
What's Next

Roadmap & future scope

This is version one. Here's where CivicAI is headed next, with engineering effort already scoped.

PHASE 02

Persistent backend & database

Move from in-session state to a real database (PostgreSQL) — ticket history, department dashboards, SLA tracking, and citizen status lookups by ticket ID.

In Design
PHASE 03

Image-based issue reporting

Let citizens upload a photo of the pothole/garbage/damage. watsonx vision tooling classifies severity directly from the image, reducing manual description.

Planned
PHASE 04

Department-side admin console

A second interface for municipal staff — view incoming tickets by department, mark resolved, attach proof-of-work, and auto-notify citizens.

Planned
PHASE 05

SMS / WhatsApp fallback channel

For citizens without smartphones or stable internet — report issues via SMS or WhatsApp, routed through the same watsonx Orchestrate pipeline.
